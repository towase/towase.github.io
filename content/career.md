# 📚 職務経歴書

## 職務要約

ソフトウェアエンジニアとして約10年のキャリアを持つ。SIerでの開発リーダー経験を経て、2020年以降はWebサービス・SaaS企業にてバックエンドを軸にフルスタックで開発に従事。複数社でテックリード・設計リードを担い、PHP→GoへのAPI移行や、iOSアプリからWebアプリへの段階的リプレイスなど、既存システムの刷新・移行をリードしてきた。

技術的負債が蓄積した環境での開発経験が豊富で、抜本的刷新ではなく段階的改善と影響範囲制御を重視する設計スタイルを持つ。大規模データ処理の方式検討・パフォーマンス改善から、開発基盤整備・チーム育成まで幅広く対応。2026年からはフリーランスとして、AIエージェントを前提とした開発プロセス（並列オーケストレーション・ガードレール設計）を実践し、スタートアップの新規プロダクト開発において基幹機能の設計・実装をリードしている。

2015年04月01日〜2026年02月17日までの職歴は、すべて正社員としての勤務です。2026年04月以降はフリーランスとして活動しています。

## フリーランス（業務委託）
2026年04月 ～ 2026年07月

### サービス：要件定義・業務設計を支援するAIネイティブB2B SaaS（スタートアップ、社名・プロダクト名非公開）

#### 概要
LLMをプロダクトの中核に据えた新規B2B SaaSの開発に業務委託として参画。TypeScriptモノレポ（pnpm + Turborepo）環境で、バックエンド・フロントエンド・AWS/GCPインフラを横断して担当。AIエージェントを前提とした開発プロセス（論点の事前確定 → 並列分解 → 統合 → ルール化）を実践し、参画3ヶ月で120本以上のPRをマージした。

#### 役割
- ソフトウェアエンジニア（業務委託）
- 権限管理・多言語化など基幹Epicの設計・実装リード

#### 業務内容
- 権限管理（RBAC）機能の設計・実装リード
- 多言語化（i18n）基盤の設計・導入・全画面展開、およびLLM出力の言語制御
- AWSインフラの運用・コスト最適化
- AIエージェント用サンドボックス実行基盤（E2B self-hosted）の運用・改善
- ISO 27017対応のアカウント削除機能の設計・実装

#### 主な取り組み・成果

- 権限管理（RBAC）の設計・実装をEpicオーナーとして一貫して主導
  - RBACフレームワーク（OPA / Casbin / Cerbos）を比較検討の上、RLS親和性・要件流動性・学習コストの観点からRDBベースの自前実装を採用。行の可視性はRLS、書き込み・機能利用の判定は単一の認可関数に集約するハイブリッド構成を設計
  - PostgreSQL enumマイグレーションのロック特性を分析し、無停止移行とメンテナンス窓での一括移行を使い分けて本番適用
  - 認可ミドルウェアの新設、ロール階層判定の共通モジュール化、ロール変更の監査ログ記録、デプロイ順序に依存しない後方互換ロールアウトまで一貫して実装

- 多言語化（i18n）基盤をゼロから設計し、全画面へ横展開
  - 海外展開要件を受け、ライブラリ選定・腐敗防止層の設計（i18nライブラリへの直接依存をlinterで制限）・Feature Flagによる段階リリースを含む基盤を導入
  - 対象を26のsub issueに分解しAIエージェント26並列でPRを量産。統合時のコンフリクト解消計画、静的監査によるハードコード文言260箇所の検出・是正まで含めて完遂
  - UIに加え、LLM生成物の出力言語制御を3系統のAI呼び出し経路すべてに実装
  - 多言語化漏れを防ぐgit hookガードレールを整備。誤検知リスクを評価し、運用ルールの合意形成を先行させてから展開

- AIエージェント前提の開発プロセスの設計・運用
  - 「論点の事前確定 → 並列分解 → 統合 → ルール化」のワークフローを確立し、最大26並列のエージェントオーケストレーションを実運用。並列生成物の統合コストやマシンリソース律速などの運用知見をチームに還元
  - 実装判断の記録・レビュー自律対応・実機UX検証などの専用スキル群を整備。静的に検査可能な規約はプロンプトではなくlinter・git hookに落とす方針を徹底
  - マルチエージェントによる敵対的検証をレビューに組み込み、AI生成コードの品質を担保

- インフラ運用・コスト最適化
  - Aurora PostgreSQLのアップグレードを検証環境で事前検証し、本番を含む全環境へ無事故で適用
  - S3 Gateway VPCエンドポイント追加によりNAT Gatewayコストを月額約900ドル削減。AWS Cost Anomaly Detectionを全環境へ展開
  - AIエージェント用サンドボックス基盤（E2B self-hosted、Nomad/Consulクラスタ）にて、テンプレートのcontent-hashベース版管理とCI自動ビルドを新規設計し、版管理不在に起因していた障害を恒久対策。30人規模の同時利用イベントに向けた容量設計とオートスケール不具合の実機検証・修正も実施

- ISO 27017対応のアカウント削除機能の設計・実装
  - 主要SaaSの削除仕様を調査の上、認証情報の物理削除と業務データの匿名化保持を組み合わせた方式を設計・実装
